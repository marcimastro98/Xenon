01What an iCUE widget can natively do

A native widget is a sandboxed HTML/CSS page with three system bridges and a set of UI controls. IT · Un widget nativo è una pagina HTML in sandbox con tre soli ponti verso il sistema.

📊 Sensors

Read hardware telemetry: temperatures, load, fan/pump speed, voltage, power, RAM timings, battery — and FPS. Read-only.

widgetbuilder.sensorsdataprovider

🎵 Media

Read song name + artist; trigger play / pause / next / previous. No album art, no source, no position.

widgetbuilder.mediadataprovider

🔗 Link

Open a URL in the system browser. This is the only "action" a widget can take on the system.

widgetbuilder.linkprovider
Hard sandbox limits. No microphone or audio-device access · no network requests · no file writes · no application launching · no system control · expressions only, not arbitrary scripting.
IT · Niente microfono/audio, niente rete, niente file, niente avvio app, niente controllo di sistema. Questo è il motivo per cui Xenon ha bisogno di un server companion.

02The complete toolbox Corsair grants you

Everything the SDK actually gives a widget developer today — and exactly when it's useful for Xenon. IT · Tutto ciò che l'SDK concede davvero, e quando serve a Xenon. Verde = già sfruttabile da Xenon.

⛔ The hard limits (what no custom widget can do)

  • No microphone / audio-device access
  • No volume control or per-app mixer
  • No sanctioned network/HTTP capability*
  • No launching applications (only "open URL")
  • No window enumeration / focus
  • No file-system writes (only localStorage)
  • No native code / FFI (no RGB SDK, no DLLs)
  • No arbitrary background processes

* The guidelines say "no network requests by default", external <script src> is blocked from the file:// origin, and there's no CORS proxy — so network isn't a capability you can build a product on. The built-in Weather, Twitch, YouTube, Volume and App Launcher widgets are Corsair's own privileged widgets — they do things the public SDK does not expose to third parties.
IT · I limiti sono il motivo per cui Xenon ha bisogno del server companion. Corsair fa Volume/App Launcher/Twitch nei propri widget, ma non li espone a noi.

03How widget settings actually work

The loop the docs never show end-to-end: you declare controls, iCUE builds the panel, and feeds the values back as globals. IT · In un widget non costruisci le impostazioni: le dichiari e iCUE genera il pannello da solo.

1

You declare a control with a <meta> tag

No settings UI to build — just describe it in the <head>. The content becomes the variable name.

<meta name="x-icue-property" content="theme" data-label="tr('Theme')" data-type="combobox" data-values="['Xenon', 'Ocean', 'Ember']" data-default="'Xenon'">
2

iCUE auto-generates the settings panel

That one tag becomes a real dropdown in iCUE's sidebar — you draw nothing.

⚙ Widget settings
Theme
Xenon ▼
3

The user picks an option

Say they choose Ocean. iCUE stores it and hands it to your widget.

4

iCUE injects it as a global variable

Named exactly after content. You don't declare it — it's just there.

// injected by iCUE, ready to read console.log(theme); // "Ocean"
5

iCUE calls onDataUpdated() on every change

Your one job: read the global and update the UI. That's the whole loop.

icueEvents = { onDataUpdated: applyTheme }; function applyTheme() { document.body.dataset.theme = theme; // "Ocean" }

06Technical notes

📌 Note personali — per te, non da mostrare se non vuoi

Perché serve un server companion (in una frase)

La sandbox del widget espone solo 3 ponti (sensori in lettura, media base, apri-link). Tutto ciò che è "potente" — microfono, audio, rete, avvio app, controllo finestre, file — è proibito. Quindi ho costruito un server Node locale che fa il lavoro vero, e il widget gli si appoggia (oggi via web/iframe).

I 4 attriti SDK da citare alla call

  • Nessuna API di sistema oltre i 3 plugin → mic/audio/rete/finestre richiedono il server esterno.
  • Validatore con motore JS datato → niente ?. né ??, codice più verboso.
  • Qt WebEngine blocca fetch → ho dovuto usare JSONP (tag <script>) per far parlare widget e server.
  • DPR frazionario (~1.8) → linee sottili sgranate; risolto con zoom calcolato in index.html.
