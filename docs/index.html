Second screen

Updates, in-app

See what’s new in a polished dialog and update in one click — download, verify and restart, with your data preserved.

Smart Home

Control your home’s lights, sensors and appliances via Home Assistant, grouped by room — live, local and lightweight, on a tile and Deck keys.

Widget SDK

Build your own widget — a sandboxed HTML tile with no network and only the data and actions you approve. Anyone can extend the dashboard.

Notification hub

Every Windows toast — WhatsApp, mail, Teams, Discord — mirrored on the dashboard, plus your Discord DMs and mentions. Read locally, private by design.

Sensor history

A History tab charts CPU/GPU temp and load and RAM over 24h / 7d / 30d — plus PC screen-time: your most-used apps and games. No AI, all local.

Context profiles

Set a page, lighting and Deck profile per activity — gaming, coding, streaming — and the dashboard switches to match, then reverts when you’re done.

Proactive moments

A game-session recap, sustained-heat alerts and a morning agenda — computed locally, no AI, and each one you can switch off.

Hands-free voice

Say “Hey Xenon” to open the assistant — no tapping. A local wake word (on-device Whisper), off by default, that never sends audio anywhere.

Native app on the Edge

Run Xenon as a full-screen Windows app that owns the Xeneon Edge — no browser, no iCUE. It self-starts, pins itself to the panel, lives in the system tray and keeps itself updated; swipe up from the bottom edge to peek at your desktop, then tap the home bar to snap back.

Game-safe touch

Touch the Edge mid-game and nothing breaks: the native app snaps the cursor back to the screen it came from and never lets a tap steal the game’s focus — no more minimized full-screen games. The stock iCUE dashboard can’t do this, nor can a browser tab: only a real native window can tell Windows how to treat it.

Developers · Widget SDK

Build your own widget.

A widget is just a folder: a manifest and an HTML page. It runs in a locked sandbox with no network, and talks to Xenon through one small, versioned bridge. Ask for the data and controls you need — the user approves them once — and your widget drives the real thing. No account, and nothing leaves the PC.

Example — keep the keyboard on-theme

Two files. The manifest asks for the chroma permission; the script reads the dashboard's accent colour and paints your Razer keyboard with it, following along whenever the theme changes.

manifest.json
{
  "api": 1,
  "id": "theme-chroma",
  "name": "Theme → Chroma",
  "entry": "index.html",
  "actions": ["chroma"]
}
widget.js
// Ask for "chroma" in the manifest, then paint the
// keyboard with the dashboard accent — live.
const send = (m) => parent.postMessage({ xenonSdk: 1, ...m }, '*');

addEventListener('message', (e) => {
  const m = e.data;
  if (!m || m.xenonSdk !== 1) return;
  if (m.type === 'init' || m.type === 'theme') {
    send({ type: 'action', id: 1, action: {
      type: 'chromaColor', device: 'keyboard', color: m.theme.accent } });
  }
});

send({ type: 'hello' });   // start the handshake
1
Request

List the streams and actions you need in manifest.json. Anything you don't ask for, you can't touch.

2
Approve

On first add, Xenon shows the user exactly what you asked for. They grant it once — and can decline.

3
Build & share

Zip the folder and send it. Anyone drops it into their widgets folder and taps Rescan. That's the whole pipeline.
